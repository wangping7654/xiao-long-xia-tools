# API使用量监控系统

## 系统概述
**目的**: 监控和控制OpenClaw API token使用量，防止意外费用
**重要性**: ⭐⭐⭐⭐⭐ (生存关键系统)
**监控对象**: deepseek/deepseek-chat模型API使用

## 监控指标

### 1. 基础指标
- **请求次数**: 每日/每周/每月API调用次数
- **Token消耗**: 输入token + 输出token总量
- **费用估算**: 基于token消耗的费用计算
- **使用趋势**: 时间分布和使用模式

### 2. 成本指标
- **每日成本**: 基于使用量的每日费用估算
- **月度预算**: ¥200/月 (目标控制)
- **成本预警**: 达到预算的50%/80%/90%时预警
- **异常检测**: 异常高使用量检测

## 监控方法

### 1. 日志分析
**日志位置**: `%TEMP%\openclaw\` 目录下的日志文件
**分析频率**: 每日分析一次
**分析内容**:
- 提取API调用记录
- 统计token使用量
- 计算费用估算

### 2. 实时监控 (待实现)
**方法**: 通过OpenClaw Gateway API监控
**频率**: 每小时检查一次
**预警**: 实时异常使用预警

### 3. 手动记录
**记录表**: 每日手动记录重要会话的token使用
**估算方法**: 基于会话长度和复杂度估算

## 费用估算模型

### DeepSeek API定价 (假设)
**输入token**: ¥0.14/1M tokens
**输出token**: ¥0.28/1M tokens
**免费额度**: 未知 (需确认)

### 估算公式
```
每日费用 = (输入token数 × 0.14 + 输出token数 × 0.28) ÷ 1,000,000
月度费用 = 每日费用 × 30
```

### 典型使用场景估算
1. **简短对话** (1000 tokens): ~¥0.00042
2. **中等对话** (5000 tokens): ~¥0.0021
3. **长文档分析** (20000 tokens): ~¥0.0084
4. **代码生成** (10000 tokens): ~¥0.0042

## 预算控制策略

### 月度预算: ¥200
**每日限额**: ¥6.67
**每周限额**: ¥46.67

### 预警阈值
- **黄色预警**: 达到月度预算的50% (¥100)
- **橙色预警**: 达到月度预算的80% (¥160)
- **红色预警**: 达到月度预算的90% (¥180)
- **紧急停止**: 达到月度预算的95% (¥190)

### 节流策略
1. **轻度节流** (黄色预警): 减少非必要使用
2. **中度节流** (橙色预警): 只执行关键任务
3. **重度节流** (红色预警): 只执行生存必需任务
4. **完全停止** (紧急停止): 暂停所有API使用
